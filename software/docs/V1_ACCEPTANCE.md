# SEEKR V1 Acceptance

Run from the SEEKR software repo root:

```bash
npm ci
npm run acceptance
```

`npm run acceptance` runs `npm run check`, `npm run bench:edge`, `npm run bench:flight`, `npm run bench:sitl`, both `npm run bench:sitl:io` fixtures, `npm run bench:dimos`, `npm run safety:command-boundary`, `npm run test:ai:local`, `npm run test:ui`, `npm run smoke:preview`, `npm run smoke:rehearsal:start`, `npm run release:checksum`, `npm run acceptance:record`, and `npm run probe:api` sequentially. The command-boundary scan writes `.tmp/safety-evidence/` and fails on obvious hardware-command regressions. The preview smoke performs the production build and then serves the built shell and API from the server process. The rehearsal-start smoke launches the operator wrapper on temporary local ports, checks API/client/source-health/readiness behavior, shuts down, and writes exact ordered `.tmp/rehearsal-start-smoke/` evidence without validating hardware. The release checksum writes JSON, SHA-256, and Markdown install-integrity evidence under `.tmp/release-evidence/`, including `.gitignore` and `.npmrc` so repository-safety policy is part of the release hash. The acceptance recorder validates the latest passing command-boundary scan and the named strict local AI smoke scenarios, including strict-AI `commandUploadEnabled: false`, saved per-case selected plan kind, validator pass, unsafe-text, and mutation fields before writing `.tmp/acceptance-status.json`, rejects acceptance output, release evidence, and safety evidence paths that escape the project root, and persists the strict AI scenario names so `/api/session` can show the latest accepted build/run evidence and scan summary without exposing secrets; the final API probe verifies that session-visible evidence and writes `.tmp/api-probe/` evidence.
`npm run test:ai:local` requires local Ollama and `llama3.2:latest` unless `SEEKR_OLLAMA_MODEL` is set to another installed model. Skip only in environments that intentionally run deterministic rules outside the acceptance gate.

Use `npm run fixtures:update` only when intentionally accepting changed deterministic scenario outputs.
